Welcome to my Text Box Generator . Lemme tell you all about how this thing works, anything that's COMPLEX will be marked in  red (like this) to make what you have to worry about more clear.

Complex Mode gives you more customizability, however it isn't as clean, and might not play well on devices like phones . Anything that uses more than one of each "element" should probably be done in complex mode.

Base Settings

There's a few basic settings, such as the width and height of the generated image, and if you want a margin around the image. If checked, before anything is generated, the image will be filled with black, including a 6px border. This will remove transparency however, so don't turn this on if you want your final image to be see through!

You can also pick the Style of the image, which'll set up a bunch of other stuff. You can pick from the built in presets, or import your own from a file of the ctbt format.  More on creating those later.

The order in which each element renders is boxes, portraits, text, and overlays , and  within those groups, higher elements are rendered first.

Box Settings

Each box will have these settings.

  • Image: You can upload an image to use for the box, though if it says [Selected], you're using the one set in the style for it.
  • Box X: The  horizontal distance of the box's left edge  from the left side of the image.
  • Box Y: The  vertical distance of the box's top edge from the top side of the image.
  • Color Type: How color changes will be applied, only to pixels of pure #ffffff,   multiplied with each pixel, or replacing  every pixel. Default is only white.
  • Color: The color the box will be, by hex color.
  • Visible: If unchecked, box will not render. Default is checked.

Portrait Settings

To begin, you'll notice the Import Character field. This allows you to upload ctbc files , which contain a character and their expressions. More on that elsewhere.

The settings for each portrait is as follows.

  • Portrait X and Y: The position of the portrait from the top left of the image, either it's center  or the top left, depending on the scaling option.
  • Expression: The expression to use! You can see a preview of it at the bottom  of each portrait box .
  • Character: The character to pick expressions from. If set to none,  the portrait will not render at all , and if set to custom, you will be given a field to upload an image . Otherwise...
  • Portrait Scaling: Lets you decide how to handle the rendering the image. In order:
  • 2X Size (Centered): Scales the image up by 2X , before centering it at the position.
  • 2X Size: Same as ^, but the position is the top left corner .
  • Centered: Like 2X Centered, but no scaling is done .
  • Unchanged: Uses the position as top left corner with no scaling .
  • 0.5X Size (Centered): Like 2X Centered, but scales DOWN instead.
  • 0.5 Size: Same as ^, with position as top left.
  • Arbitary Scaling (Centered): Lets you set the scaling factor for each axis,  before centering.
  • Arbitary Scaling: Same, but with no centering.
  • Arbitary Size (Centered): Lets you set the width and height in pixels , and squishes and stretches to fit before centering.
  • Arbitary Size: Same, no centering.
  • Scale Evenly to Fit: You set the width and height, and it does math to scale both up by the same factor , so the width or height is the same as set. Centered.
  • Outline: Generates a 1px black outline around the portrait if checked.
  • Color Type: Same settings as with the box.
  • Color: Color in hex.

Text Options

Text is what you've come here to make, so of course it's got options AND commands !

  • Font: Choose the font to use, you can pick from the presets, or upload an image to use as the font. More on that elsewhere.
  • Monospaced: If custom, then you can choose if it's the same spacing between each letter , or if unchecked, defined in a file . More elsewhere.
  • Text X and Y: Top left corner of text. Duh.
  • Portrait X and Y Offset: How much to move the text IF a portrait is being used (not set to none).
  • Outline: If checked, each letter has a 1px black outline . Good for complex boxes.
  • Dark World: Adds a blue drop shadow if checked. Not compatable with Outline .
  • Auto-Linebreak: Attempts to automatically line break if a certain width is reached , with * awareness .  Can be buggy .
  • Color Blend: Same options as boxes.

The text itself is... the text itself, duh. You write letters in. However, it has a few commands you can insert mid text ! 

  • \n: Linebreak.
  • \#(hex): Changes color from then onward.
  • \x(int)]: Moves the text position that amount to the right.
  • \y(int)]: Moves the text position that amount downwards .

Overlay Options

Overlays are the last element type, going above everything else.

  • Overlay: Pick from a preset image, or upload your own.
  • X and Y: The top left position of the image.
  • Width and Height: How many pixels to draw from the top left . -1 is the whole axis .
  • Visible: If unchecked, will not render .

Every Element

In complex mode, you'll see there can be multiple of each type. In simple mode, you only see the highest element of each type. You'll also have a button that will add a new element of said type, appended to the bottom .

In addition, every element will have the name setting. This property is purely cosmetic , and used for specifying the purpose of the element within the style.

Alongside that, every element has three buttons at their bottom. In order, the ^ button will move it up by one (rendered before any lower elements of the same type), the x button will remove the element entirely, and the v button will move the element down by one (rendered after any higher elements of the same type.)

Generating

Generating the box is as simple as clicking the button! Just know, it won't update until you click, and it could take a few clicks if everything doesn't render out!

Box Stacking

You can stack boxes into one image! Click Add to Stack to append the currently generated box to the stack. Each box will be added without a margin,  though it cannot readd transparency !

Refresh Stack will regenerate the stack image, and Clear Stack removes all the boxes.

Each box has three buttons next to it. In order, the ^ button will move the box up one, the x button removes it from the stack, and the v   button will move the box down one.

Boxes are put into the stack from the top most box to the bottom most , and in reading order (ie, go left to right, then down one, repeat). The amount of times it'll go right before going down one and to the left is determined by the Stack Width .

If the boxes aren't all the same size, it'll do it's best to center them in their spots. If Margin 's on, there'll be a 6px black margin between each box (alongside the 6px border for the whole image).

Custom Styles

To save a custom style, just click the Save Custom Box button next to the Generate Box button, and it'll make a ctbt file with all your settings (aside from portrait contents and the such).
